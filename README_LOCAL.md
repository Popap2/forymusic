# WWmusic - Локальный запуск

Инструкция по запуску проекта локально без внешних сервисов (Supabase, Render и т.д.).

## Что нужно

- Node.js (версия 14 или выше)
- npm

## Быстрый старт

### 1. Установка зависимостей

Откройте терминал в папке проекта и выполните:

```bash
npm install
```

Это установит все необходимые пакеты:
- `express` - веб-сервер
- `sqlite3` - локальная база данных
- `multer` - загрузка файлов
- `bcrypt` - хеширование паролей

### 2. Запуск сервера

```bash
npm start
```

Или:

```bash
node server.js
```

Сервер запустится на порту 3000 (или на том, который указан в переменной окружения `PORT`).

### 3. Открытие в браузере

Перейдите по адресу:

```
http://localhost:3000
```

## Как это работает локально

### База данных

- Используется **SQLite** (файл `music.db` в корне проекта)
- База данных создаётся автоматически при первом запуске
- Все данные (пользователи, треки) хранятся в этом файле
- При удалении `music.db` база создастся заново (пустая)

### Загрузка файлов

- Аудио-файлы сохраняются в папку `uploads/` в корне проекта
- Папка создаётся автоматически при первом запуске
- Файлы доступны по адресу `/uploads/имя_файла.mp3`

### Пользователи

- Регистрация и вход работают через локальную базу данных
- Пароли хешируются через `bcrypt` перед сохранением
- Лайки и плейлисты сохраняются в базе данных

### Владельцы (админы)

По умолчанию админ-права имеют:
- `zilajrik7@gmail.com`
- `ilove@you.com`

Для управления треками (добавление, редактирование, удаление) нужен **admin-пароль**.

По умолчанию admin-пароль: `dev-admin-password`

Чтобы изменить его, создайте файл `.env` в корне проекта:

```
ADMIN_PASSWORD=ваш-секретный-пароль
```

- Прослушивать треки
- Ставить лайки
- Создавать плейлисты

### Владельцы (админы)

## Изменение порта

По умолчанию сервер запускается на порту 3000.

Чтобы изменить порт, создайте файл `.env`:

```
PORT=8080
```

Или установите переменную окружения перед запуском:

**Windows (cmd):**
```cmd
set PORT=8080
node server.js
```

**Windows (PowerShell):**
```powershell
$env:PORT=8080
node server.js
```

**Linux/Mac:**
```bash
PORT=8080 node server.js
```

## Решение проблем

### Ошибка "Cannot find module"

Убедитесь, что выполнили `npm install` в папке проекта.

### Порт уже занят

Измените порт через переменную окружения `PORT` (см. выше) или остановите процесс, который использует порт 3000.

### База данных не создаётся

Проверьте права на запись в папку проекта. Убедитесь, что у вас есть права на создание файлов.

### Файлы не загружаются

Проверьте, что папка `uploads/` существует и доступна для записи. Она создаётся автоматически, но если её нет - создайте вручную.

## Остановка сервера

Нажмите `Ctrl + C` в терминале, где запущен сервер.

## Важно

- Все данные хранятся локально в файле `music.db` и папке `uploads/`
- При удалении этих файлов все данные будут потеряны
- Для продакшена рекомендуется использовать внешнюю базу данных (PostgreSQL) и облачное хранилище
